Midiendo el rendimiento de un detector de fraude¶

Las actividades fraudulentas son constantemente perseguidas, por lo que los defraudadores necesitan ser creativos e inventar nuevas formas de llevar a cabo sus fraudes. Adems, afortunadamente, las operaciones fraudulentas son relativamente escasas, lo que nos lleva a contar con pocos casos positivos para entrenar el modelo. Dicho de otro modo, nos enfrentamos a un problema altamente desequilibrado, lo cual dificulta el entrenamiento del modelo, as como su evaluacin.

Consideremos un modelo trivial que clasifica todos los casos como negativos (operaciones legtimas). Podemos simular las predicciones de este modelo creando un vector de predicciones de todo ceros:

In [ ]:
dumbpreds = [0] * len(test)

Midamos el porcentaje de acierto de este modelo sobre el conjunto de test

In [ ]:
from sklearn.metrics import accuracy_score

accuracy_score(test["Class"], dumbpreds)
Out[ ]:
0.9531071292413267

Deberas haber obtenido en torno a un 95% de acierto, ya que la gran mayora de casos son negativos. A pesar de eso, este modelo es totalmente intil como detector de fraudes! Por tanto, necesitamos una mtrica mejor.

Una mtrica que funciona bien para problemas muy desbalanceados es el rea bajo la curva ROC, o AUC. En scikit-learn esta mtrica est disponible, y podemos probarla para comprobar que el rendimiento de este modelo es realmente malo

In [ ]:
from sklearn.metrics import roc_auc_score

roc_auc_score(test["Class"], dumbpreds)
Out[ ]:
0.5

Un AUC del 50% indica que el modelo no es mejor que lanzar predicciones aleatoriamente. Si evaluramos un modelo en el que las predicciones de probabilidad de la clase fraude fueran algo ms altas para los casos realmente fraudulentos que para los casos legtimos, veramos cmo el AUC produce mayores valores. El caso ptimo para esta mtrica es un modelo en el que todos los casos de fraude son predichos con una mayor probabilidad de fraude que todos los casos de operaciones legtimas.

Detector de fraude no supervisado¶

Dado que apenas tenemos datos de fraude, puede tener sentido empezar construyendo un modelo no supervisado.

Usando solo los datos de train, y sin emplear la variable Class, construye un modelo de tipo IsolationForest para deteccin de anomalas. Mide el rendimiento del modelo sobre el conjunto de test, usando la mtrica AUC.

Ten en cuenta los siguientes puntos para construir un buen modelo:

  • Puedes estimar el parmetro contamination como la proporcin de datos fraudulentos del conjunto de entrenamiento.
  • Por defecto IsolationForest emplea pocos rboles. Asegrate de probar con diferentes nmeros de rboles.
  • La mtrica AUC necesita recibir las probabilidades de clase fraude para funcionar correctamente. No es posible obtener probabilidades de un IsolationForest, pero puedes hacer uso de su funcin decision_function para obtener scores normalizados (profundidades medias en el rbol), las cuales pueden ser negadas para obtener valores que pueden interpretarse como probabilidades de clase fraude (no normalizadas).
Si has entrenado el modelo correctamente, deberas obtener en torno a un 94% de AUC.

features = [col for col in train.columns if col != 'Class']
Xtrain = train[features].values
Xtest = test[features].values

Ytrain=train['Class'].values
Ytest=test['Class'].values

percent_fraud = sum(train.Class == 1)/train.shape[0]
In [ ]:
from sklearn.ensemble import IsolationForest
from sklearn.metrics import roc_auc_score

def trainscore_isolation_forest(X, Xte, yte, estimator):
    estimator.fit(X)
    score = np.negative(estimator.decision_function(Xtest))
    return roc_auc_score(yte, score)

trange = [1, 10, 20, 50, 100, 500, 1000]

aucs = {
        ntrees :
        trainscore_isolation_forest(Xtrain, Xtest, Ytest, IsolationForest(n_estimators = ntrees, contamination=percent_fraud))
        for ntrees in trange
}

isolation_forest_df = pd.DataFrame({        
        "ntrees" : [ntrees for ntrees in aucs],
        "auc" : [aucs[ntrees]  for ntrees in aucs]
    })
